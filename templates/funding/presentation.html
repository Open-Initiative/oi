{% trans "What is cofunding? Is it different from crowdfunding?" %}

{% trans "

Open Funding is different from usual crowdfunding platforms. Cofunding provides a sustainable business model for free software and enables users to contribute and get their voice heard. It takes a professional approach to an open contribution world, and it makes the software more sustainable by helping professionals to keep working on it.

" %}

{% trans "A business model for free software developers! Really?" %}

{% trans "

Open Funding is the best business model free software developers can have. Projects are funded feature by feature, thus reducing the risk associated with the development, and making sure the funds serve the user needs. Plus as long as developers add features, they get funded, so projects can last for as many years as they need!

Developers are committed to their results and users validate the job when it is done. We also provide real assistance to developers on their communication. So we’re really talking about professional development for your software.

" %}

{% trans "So as a user I can now contribute to my software!" %}

{% trans "

Yes! It is free software, so you can just make it yours! Give your opinion and fund the features you really need. Open Funding is the right place for you to join the community and make your software even better for your needs!

" %}

{% trans "What makes Free software so efficient?" %}

{% trans "

Free software has its origins in the 1980s, when some computer users wanted to be able to change the software that run these computers. So created software that the users are allowed to modify and redistribute. Many people started to contribute to the software, and it quickly became clear that this system had many advantages. Today free software appears as a credible model that has proved its efficiency. By reusing what has been done by others, developers can quickly create new applications and contribute to the common good that free software is. Plus they get contributions from a larger community: user feedbacks and bug reports, patches for bugfixes or new features, translations…

" %}

{% trans "If it is Free software, why should it have a price?" %}

{% trans "

Well, as Stallman said, it is free as in “free speech”, not as in “free beer”. Free software developers need to earn money, as everybody else. Today many developers sell services around their software, but these revenues are not directly related to the development work itself.

Open Funding allows you to contribute by funding the features you need to see added to the free software you use. And once the developer has been paid for his work, he can really let it go free!

" %}

{% trans "Why did I think contributing to Free software was none of my business?" %}

{% trans "

No idea! Today, software is a foundation of our society. There is no machine running without software. The software is simply a recipe that makes many areas of our activities run. At Open Initiative, we believe that this is everyone's business, and it should not be left to financial interests or some enlightened technicians. Open Funding allows you to participate in the construction of free software if you are a developer who wants to live of your work or a user who wants software that finally fits your needs.

" %}

All projects that are free software can be funded on OpenFunding. You could either submit your free software project or contact us if you’re looking for a free software project idea to work on, or if you have an idea you want to discuss. Come talk to us! We’ll make great software together.

{% trans "How does it work?" %}

{% trans "

The developers propose their free software projects on the platform with a list of features and the amount of money they need to develop each of them. Users will fund with the amount they want according to their needs. You can either fund any of the proposed feature or you can choose to back the the whole project. The developer then decides the feature for which the money can be most useful.

" %}

{% trans "What happens once the feature is funded?" %}

{% trans "

Once the requested amount is reached, the developer can start working! He delivers the feature when he’s done, and users then have 14 days to validate the result or request fixes. The developer gets paid after each user validation.

This way Open Funding secures the conclusion of the project to the users and the payment to developers once the work is done.

" %}

{% trans "How do you guys make a living?" %}

{% trans "

Open Funding takes a fee limited to a 5% commission added to the amount requested to the developer once the feature is funded.

" %}

{% trans "Is OpenFunding involved in helping the developer for description of the project and fixing on the cost? Is there an openness/transparency when it comes to propose the cost structure of the feature?" %}

{% trans "

If you have a free software project, you propose it on the platform with a list of its features. You explain your project and its features in the creation page, and for each feature you determine the amount required for their development. You can add several features for each project. We add our commission to this sum: 5% exclusive of taxes.

" %}

{% trans "Is there a time limit for the campaigns? How long can I run the project on OpenFunding?" %}

{% trans "

Open Funding is designed to make projects sustainable. So the projects can last as long as users need new features. Currently there is no limit to single feature campaigns because several features can be funded at the same time, but that might change in the future.

" %}

{% trans "I created a project, what now? Can I get help for my campaigns?" %}

{% trans "

Before publishing your project on the platform we will contact you by email and then by telephone. We wish to establish contact with project owners to learn a bit more from them directly, and assist them in the way they will promote the project.

Once we have validated this together, it’s time to share your project with the community. Each developer is free to communicate the way they wish and transmit the information to users to promote and fund the project. However, we are always available to guide you through this process.

" %}

{% trans "You can contact us anytime" %}: contact@open-initiative.com

{% trans "I gathered the funds, what now?" %}

{% trans "

When a feature reaches the required amount, you can start the development. The development time is specified in the description of evolution. At any time, you can communicate with funders on the platform to discuss its progress.

Once the development is done, you deliver the product. Each developer determines with his funders the best way to deliver the result. The product can be integrated into the next version of the software, but it can also be a fork, be delivered as a patch, etc., as long as it is clearly stated from the start.

Users have 14 days to validate the feature or request corrections to the feature. The feature is automatically validated after 14 days if the user hasn’t answered. Each time a user validates the development, the amount is paid to the developer.

Open Funding receives 5% commission excluding taxes of the feature’s total cost.

" %}

{% trans "What happens if the project is unable to raise the funds?" %}

{% trans "

The contract can be changed at any time during the funding phase. So you can always agree on a new price, and even agree on the amount actually raised. But you can also modify the proposition so that it covers better the user needs, or adapt your communication strategy. In any case, don’t hesitate to come discuss it with us!

" %}

{% trans "Why are there two options: ‘Order the feature’ & ‘Back the Project’ ?" %}

{% trans "

You can either fund any of the proposed feature or you can choose to fund the software; and then the developer decides for which feature he can use the money.

Order the feature: By contributing to this order you will be delivered the professional quality feature you always wished for. Once the feature is developed, you will receive an email to test it and give your validation.

Back the project: By Co-funding this project you enable the developer to choose what feature to develop, so that the software on the whole makes better sense for you. Once the development is done, you will receive an email to test the new feature and give your validation.

" %}

{% trans "What is ‘your credit’ for?" %}

{% trans "

You have credit in your account that allows you to fund projects on the site. You can increase your credit at any time in the My Account section. This way you help funding your favorite projects with the amount according to your interests. You can modify you credit at any time.

Our secured system allows you to keep your account and funding secure.

" %}

{% trans "What if the feature I paid for doesn’t reach the required amount? is my money lost?" %}

{% trans "

No money lost! The developer will maybe adapt the amount or the feature itself to make it happen. But you too can help! Talk to other users you know, spread the word about that great new feature that would make the software so nice.

In any case, you can always cancel your order of the feature until the developer has started the development. You can then use the money on another feature, or get a refund.

" %}
